# 双语字幕校对 (Bilingual Subtitle Proofreader)

## 核心原则

### 中文校对规则
1. **保留原意** - 中文已是录制口播版本，不可更改内容含义
2. **修正识别错误** - 仅修正语音识别导致的问题：
   - 同音错字（如"cloud扣" → "Claude Code"）
   - 专有名词拼写（如"APIK" → "API Key"、"Mini Max" → "MiniMax"）
   - 明显的语音转文字错误
3. **保持口语风格** - 不要将口语改成书面语

### 英文优化规则
1. **地道表达** - 重写为自然的英语，而非逐字翻译
2. **YouTube风格** - 适合视频字幕的简洁、口语化表达
3. **技术术语准确** - 保持技术词汇的标准英文写法
4. **语气一致** - 保持友好、专业的科技博主风格

### 时间轴与断句规则
1. **时间轴对应** - 中英文必须与原时间轴一一对应
2. **合理断句** - 如原断句不合理，可适当调整断句位置
3. **保持同步** - 确保中英文字幕数量和顺序完全匹配

## 工作流程

### Step 1: 读取字幕文件
读取待校对的 .srt 文件，识别中文部分和英文部分的分界。

### Step 2: 中文校对
逐条检查中文字幕：
- 识别并修正同音错字
- 修正专有名词（产品名、技术术语、品牌名）
- 检查是否有明显的语音识别错误

### Step 3: 英文优化
逐条优化英文字幕：
- 重写为地道的英语表达
- 保持与中文内容对应
- 使用适合YouTube视频的语气

### Step 4: 断句检查
检查中英文断句是否合理：
- 一句话被不自然地切断时，考虑调整
- 确保每条字幕长度适中（便于阅读）

### Step 5: 输出校对结果
输出完整的校对后字幕文件，保持原有的SRT格式。

## 示例

### 中文校对示例

**原文（识别错误）：**
```
在cloud扣里面使用它
```

**校对后：**
```
在 Claude Code 里面使用它
```

**原文（专有名词错误）：**
```
拿到APIK
```

**校对后：**
```
拿到 API Key
```

### 英文优化示例

**原文（机翻风格）：**
```
They are all generated by a prompt word in the cloud code.
```

**优化后（地道表达）：**
```
I built all of these with just a single prompt in Claude Code.
```

**原文（生硬翻译）：**
```
The operation is simple.
```

**优化后（自然口语）：**
```
It's super easy to set up.
```

## 输出格式要求

校对完成后，输出完整的 .srt 文件内容，格式如下：

```
1
00:00:00,000 --> 00:00:02,000
中文字幕内容

2
00:00:02,000 --> 00:00:04,000
下一条中文字幕

...

[英文部分从新的序号开始]

N
00:00:00,000 --> 00:00:02,000
English subtitle content

N+1
00:00:02,000 --> 00:00:04,000
Next English subtitle
```

## 注意事项

1. **不要过度修改** - 中文只改错误，不改风格
2. **保持人称一致** - 视频通常是第一人称"我"
3. **技术内容准确** - 确保技术描述正确
4. **文化适配** - 英文可以适当调整以符合英语观众习惯
5. **字幕长度** - 每条字幕不宜过长，便于观看时阅读
